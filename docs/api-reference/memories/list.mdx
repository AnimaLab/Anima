---
title: 'List Memories'
description: 'Retrieve memories with pagination and filtering'
---

## GET /api/v1/memories

<ParamField header="X-Anima-Namespace" type="string" required>
  Target namespace
</ParamField>

## Query Parameters

<ParamField query="offset" type="integer" default="0">
  Pagination offset
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Number of memories to return
</ParamField>

<ParamField query="status" type="string">
  Filter by memory status
</ParamField>

<ParamField query="memory_type" type="string">
  Filter by memory type: `raw`, `reflected`, `deduced`, or `induced`
</ParamField>

<ParamField query="category" type="string">
  Filter by semantic category (e.g. `preference`, `identity`, `routine`)
</ParamField>

<ParamField query="cursor" type="string">
  Cursor for keyset pagination. Pass the `next_cursor` value from a previous response to get the next page. When present, `offset` is ignored. Provides constant-time page fetches regardless of depth.
</ParamField>

## Response

Returns a paginated list of memory objects. When using cursor pagination, the response includes `next_cursor` (null when there are no more results).
