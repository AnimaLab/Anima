## Install

Pick whichever method gets you running fastest.

<Tabs>
  <Tab title="Download binary">
    Download the latest binary for your platform from [GitHub Releases](https://github.com/AnimaLab/Anima/releases). No Rust toolchain needed.

    ```bash
    # macOS (Apple Silicon)
    curl -L https://github.com/AnimaLab/Anima/releases/latest/download/anima-server-latest-aarch64-apple-darwin.tar.gz | tar xz
    cd anima-server-* && ./anima-server
    ```

    ```bash
    # Linux (x86_64)
    curl -L https://github.com/AnimaLab/Anima/releases/latest/download/anima-server-latest-x86_64-unknown-linux-gnu.tar.gz | tar xz
    cd anima-server-* && ./anima-server
    ```

    The archive includes the binary, `config.default.toml`, and service templates.
  </Tab>
  <Tab title="Docker">
    ```bash
    docker run -p 3000:3000 -v ./anima.db:/app/anima.db ghcr.io/animalab/anima:latest
    ```
  </Tab>
  <Tab title="Build from source">
    ```bash
    cargo build --release -p anima-server
    ./target/release/anima-server
    ```
  </Tab>
</Tabs>

Anima starts on `http://localhost:3000`. On first run it downloads the local embedding model (~200 MB) automatically. The web UI is served at the same address.

## Store and Search

<Steps>
  <Step title="Store a memory">
    ```bash
    curl -X POST http://localhost:3000/api/v1/memories \
      -H "Content-Type: application/json" \
      -H "X-Anima-Namespace: default" \
      -d '{"content": "I love hiking on weekends.", "tags": ["preference"]}'
    ```
  </Step>
  <Step title="Search">
    ```bash
    curl -X POST http://localhost:3000/api/v1/memories/search \
      -H "Content-Type: application/json" \
      -H "X-Anima-Namespace: default" \
      -d '{"query": "What does the user like to do on weekends?", "limit": 5}'
    ```
  </Step>
</Steps>

Store and search work with zero configuration — no LLM needed.

You can also specify a semantic category and source:

```bash
curl -X POST http://localhost:3000/api/v1/memories \
  -H "Content-Type: application/json" \
  -H "X-Anima-Namespace: default" \
  -d '{
    "content": "I prefer dark mode in all my editors.",
    "category": "preference",
    "source": "user_stated"
  }'
```

## LLM Setup

The `/ask` and `/chat` endpoints need an LLM backend. By default Anima expects Ollama:

```bash
ollama serve
ollama pull qwen3.5:4b
```

To use a cloud API instead (OpenAI, Groq, Together, etc.), edit `config.default.toml`:

```toml
[llm]
base_url = "https://api.groq.com/openai/v1"
model = "qwen/qwen3-32b"
api_key = ""  # or set OPENAI_API_KEY env var
```

<Info>
  Without an LLM, `/ask` degrades gracefully — it returns retrieval results directly with `"degraded": true` instead of failing.
</Info>

## Ask and Chat

<Steps>
  <Step title="Ask a question">
    ```bash
    curl -X POST http://localhost:3000/api/v1/ask \
      -H "Content-Type: application/json" \
      -H "X-Anima-Namespace: default" \
      -d '{"question": "What pet does the user have?"}'
    ```

    Response:
    ```json
    {
      "answer": "The user has a golden retriever named Max.",
      "memories_referenced": [{"id": "...", "content": "I have a golden retriever named Max.", "score": 0.89}],
      "elapsed_ms": 1234
    }
    ```
  </Step>
  <Step title="Chat with memory context">
    ```bash
    curl -X POST http://localhost:3000/api/v1/chat \
      -H "Content-Type: application/json" \
      -H "X-Anima-Namespace: default" \
      -d '{"message": "What do you know about my dog?"}'
    ```
  </Step>
</Steps>

## Embedding Backend

By default, Anima uses a local ONNX embedding model — no API calls required. To use a cloud provider instead:

```toml
[embedding]
backend = "openai_compat"
api_base_url = "https://api.cohere.com/compatibility/v1"
api_model = "embed-v4.0"
api_key = ""  # or set EMBEDDING_API_KEY env var
dimension = 1024
```

Works with OpenAI, Cohere, Voyage, Together, vLLM, or any OpenAI-compatible endpoint.

<Warning>
  Switching embedding backends on an existing database produces incompatible vectors. Start with a fresh database when changing backends.
</Warning>
